# SelFlow Monitor

Рабочий прототип системы мониторинга и прогнозирования риска селевых потоков для рек Баксан, Малка, Черек и Чегем.

Основная демонстрационная версия находится в папке `node-dashboard`. Она сделана как локальное Node.js веб-приложение с авторизацией, русским интерфейсом, формой ввода параметров, графиками, журналом прогнозов и интерактивной 3D-поверхностью риска.

## Возможности

- серверная авторизация, проверка текущей сессии и демо-регистрация пользователя;
- личный кабинет оператора и кнопка выхода из системы;
- выбор одной из четырех рек: Баксан, Малка, Черек, Чегем;
- ввод осадков, температуры, влажности, расхода воды, снега и сейсмики;
- расчет процента риска и уровня опасности;
- сохранение новых наблюдений через локальный API;
- панель мониторинга с карточками, графиками и журналом прогнозов;
- страница базы данных с понятной схемой связей и последними записями;
- страница исторических селевых событий;
- интерактивная 3D-поверхность риска, которую можно вращать мышью;
- переключение аналитики по периодам: день, неделя, месяц;
- SVG-ярлык сайта `public/favicon.svg` для вкладки браузера.

## Требования

- Node.js 18 или новее;
- npm, который устанавливается вместе с Node.js;
- современный браузер: Chrome, Edge, Firefox.

Python нужен только для старой Streamlit-версии. Для красивого Node.js-интерфейса Python не требуется.

## Установка и запуск на Windows

1. Установите Node.js LTS с сайта [nodejs.org](https://nodejs.org/).
2. Откройте PowerShell.
3. Перейдите в папку проекта:

```powershell
cd C:\Users\dzara\Documents\Диплом\SelFlow-Monitor\node-dashboard
```

4. Запустите приложение:

```powershell
npm start
```

5. Откройте в браузере:

```text
http://localhost:3000
```

Если порт `3000` занят, можно указать другой:

```powershell
$env:PORT=3001
npm start
```

## Установка и запуск на Linux

1. Установите Node.js и npm:

```bash
sudo apt update
sudo apt install nodejs npm
```

2. Проверьте версии:

```bash
node -v
npm -v
```

3. Перейдите в папку проекта:

```bash
cd ~/SelFlow-Monitor/node-dashboard
```

4. Запустите приложение:

```bash
npm start
```

5. Откройте:

```text
http://localhost:3000
```

Для другого порта:

```bash
PORT=3001 npm start
```

## Данные для входа

На экране авторизации можно использовать демо-данные:

```text
Логин: operator
Пароль: 1234
```

Также доступна вкладка `Регистрация`. В прототипе регистрация демонстрационная: пользователь заполняет поля и попадает в рабочую панель.

Проверка авторизации выполняется через локальный API:

- `POST /api/auth/login` - вход по логину и паролю;
- `POST /api/auth/register` - регистрация нового оператора;
- `GET /api/auth/me` - проверка действующей сессии;
- `POST /api/auth/logout` - выход из системы.

Данные мониторинга (`/api/bootstrap`) и сохранение прогноза (`/api/observations`) закрыты: без действующего токена сервер возвращает `401`.

## Структура проекта

```text
node-dashboard/
  server.js          локальный Node.js сервер и API
  package.json       команды запуска
  public/

ml_model.py         ML-модель для Python-версии
```

## Проверка работоспособности

Откройте `http://localhost:3000`.

Проверьте сценарий:

1. Войти через форму авторизации или зарегистрироваться.
2. Открыть `Личный кабинет` в правом верхнем углу и проверить данные оператора.
3. Открыть `Ввод данных`.
4. Выбрать реку и ввести параметры.
5. Нажать `Рассчитать и сохранить`.
6. Проверить новую запись на страницах `Панель мониторинга` и `База данных`.
7. Открыть `Аналитика` и повернуть 3D-поверхность риска мышью.
8. Нажать `Выйти` и убедиться, что приложение вернулось на экран авторизации.

## Примечание для диплома

Node.js-версия является рабочей демонстрационной программой. Она показывает пользовательский интерфейс, расчет риска, сохранение наблюдений, визуализацию временных рядов, журнал прогнозов и интерактивную поверхность риска.

Тяжелые компоненты вроде Kafka, Flink, TimescaleDB и Kubernetes можно описывать в дипломе как промышленное развитие системы, но для защиты удобнее показывать локальную Node.js-версию.
